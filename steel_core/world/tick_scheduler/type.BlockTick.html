The block or fluid type this tick targets.

§pos: BlockPos

The block position to tick.

§delay: i32

Remaining delay in game ticks. Decremented each tick; fires when <= 0.

§priority: TickPriority

Execution priority (lower = fires first within the same game tick).

§sub_tick_order: i64

Monotonic counter for stable ordering within the same priority. Newly scheduled ticks get positive values from World::sub_tick_count. Loaded ticks get negative values to execute before newly scheduled ones.
