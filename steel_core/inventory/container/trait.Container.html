Expand description

Something that contains items. I also use container interchangeably with inventory as they mean approximately the same thing. But inventory could also refer to the player’s inventory. Example: PlayerInventory, Chest, Temporary Crafting Table

Returns the maximum stack size for a specific item in this container.

fn still_valid(&self) -> bool

Returns true if the player can still interact with this container.

This is used to validate that:

  • The container still exists (e.g., chest block hasn’t been destroyed)
  • The player is within interaction range
  • Any other conditions for valid interaction

The default implementation always returns true (e.g., for player inventory). Block-based containers should override this to check block existence and distance.

Based on Java’s Container.stillValid(Player).

fn add(&mut self, stack: &mut ItemStack) -> bool

Tries to add an item to the container.

First tries to stack with existing matching items, then tries empty slots. Returns true if the entire stack was added, false if some or all couldn’t fit. The passed stack is modified to contain any remaining items.

Based on Java’s Inventory.add(ItemStack).
