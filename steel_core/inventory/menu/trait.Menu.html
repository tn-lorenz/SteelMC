fn quick_move_stack( &mut self, guard: &mut ContainerLockGuard, slot_index: usize, player: &Player, ) -> ItemStack

Handles shift-click (quick move) for a slot.

Returns the item that was originally in the slot (before any move occurred), or empty if nothing was moved.

Based on Java’s AbstractContainerMenu::quickMoveStack.

fn can_take_item_for_pick_all( &self, _carried: &ItemStack, _slot_index: usize, ) -> bool

Returns true if the item can be taken from the slot during pickup all. Override to prevent pickup from certain slots (like crafting result).

Source

fn removed(&mut self, player: &Player)

Called when the menu is closed/removed. Override to handle cleanup like returning crafting grid items to the player. The default implementation clears the carried item by dropping it.

Source

fn clicked( &mut self, slot_num: i16, button: i8, click_type: ClickType, has_infinite_materials: bool, player: &Player, )

Handles a click action in this menu. Based on Java’s AbstractContainerMenu::clicked and doClick.

has_infinite_materials should be true if the player is in creative mode.

fn do_quick_move(&mut self, slot_num: i16, player: &Player)

Handles quick move (shift-click). Based on Java’s AbstractContainerMenu::doClick for ClickType.QUICK_MOVE.

Source

fn do_swap(&mut self, slot_num: i16, button: i8, player: &Player)

Handles swap (number keys to swap with hotbar). button is the hotbar slot (0-8) or 40 for offhand.

Based on Java’s AbstractContainerMenu::doClick for ClickType.SWAP.

Source

fn do_pickup_all(&mut self, slot_num: i16, button: i8, player: &Player)

Handles pickup all (double-click). Collects matching items from all slots into the carried stack. Based on Java’s AbstractContainerMenu::doClick for ClickType.PICKUP_ALL.
