Expand description

A slot is a view into a single position in a container. Slots require a ContainerLockGuard to access items, ensuring proper locking.

Returns the maximum stack size for this slot.

fn set_by_player( &self, guard: &mut ContainerLockGuard, stack: ItemStack, _previous: &ItemStack, )

Sets the item in this slot, triggered by a player action.

This is called when a player directly places or swaps an item in a slot. The previous parameter contains the item that was in the slot before.

Subclasses can override this to trigger events like equipment change sounds. The default implementation just calls set_item.

fn allow_modification(&self, guard: &ContainerLockGuard) -> bool

Returns true if partial removal is allowed from this slot.

For normal slots: may_pickup() && may_place(current_item) For result slots: false (must take the full stack)

Source

fn get_max_stack_size_for_item( &self, guard: &ContainerLockGuard, stack: &ItemStack, ) -> i32

Returns the maximum stack size for a specific item in this slot.

Takes the minimum of the slot’s max stack size and the item’s max stack size.

Source

fn remove(&self, guard: &mut ContainerLockGuard, amount: i32) -> ItemStack

Removes up to amount items from this slot and returns them.

Source

fn try_remove( &self, guard: &mut ContainerLockGuard, amount: i32, max_amount: i32, ) -> Option<ItemStack>

Tries to remove items from this slot with validation.

Returns Some(items) if removal succeeded, None otherwise. If allow_modification() is false and max_amount < item.count, returns None (forcing full stack pickup for result slots).

Source

fn on_take( &self, _guard: &mut ContainerLockGuard, _stack: &ItemStack, _player: &Player, ) -> Option<ItemStack>

Called when an item is taken from this slot. Returns any remainder items that couldn’t be placed back (e.g., crafting remainders).

Source

fn safe_take( &self, guard: &mut ContainerLockGuard, amount: i32, max_amount: i32, player: &Player, ) -> ItemStack

Safely takes items from this slot with all checks and callbacks.

This combines try_remove and on_take into a single operation.

Returns the items taken (empty if nothing could be taken).

Source

fn is_fake(&self) -> bool

Returns true if this is a “fake” slot (like crafting result). Fake slots don’t persist items and are virtual views.
